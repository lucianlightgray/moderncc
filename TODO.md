# C11 Conformance TODO

Goal: implement **every detail** of ISO/IEC 9899:2011 (C11), tracked here and
backed by regression tests across all targets (x86_64, i386, ARM, AArch64,
RISC-V 64), platforms (ELF/PE/Mach-O), and both glibc and musl, until every
feature is implemented and every test passes.

Reference: **N1570** (the C11 working draft; clause numbers below cite it).
Each item records: the standard requirement, the current `mcc` status (with how
it was verified ‚ÄĒ `file:line` or a test result), and the precise gap.

Default standard in `mcc` is **C99** (`cversion=199901`, `src/libmcc.c:856`);
C11 is selected with `-std=c11`. Many items below were verified at `-std=c11`.

Severity tags:
- **[BUG]** ‚ÄĒ wrong codegen, crash, or rejects-valid C11 (silent miscompile or hard failure).
- **[FEATURE]** ‚ÄĒ a C11 feature not implemented at all.
- **[DIAG]** ‚ÄĒ a required constraint violation that is accepted with no diagnostic (C11 requires "a diagnostic"); or a diagnostic that is wrong/misleading.
- **[OPT]** ‚ÄĒ optional in C11 (a conforming implementation may omit it); listed for the "implement everything" goal.

Legend: `[ ]` not started ¬∑ `[~]` partial ¬∑ `[x]` done+tested.

ALSO DONE: **float** atomic RMW (`_Atomic float`/`double`, any op) ‚ÄĒ routed through the same CAS loop, with the `desired` value read from its slot as an **exact-size integer** so it is passed in a GP register (the helper ABI), which also bit-reinterprets the float correctly. ALSO DONE: the in-language plain *read* of a wider-than-word `_Atomic` integer scalar (`y = g;`, `g + 1`, `return g`, `foo(g)`, `if (g)`) ‚ÄĒ previously two non-atomic word loads on a 32-bit target. The hook is in `gen_cast` (`gen_atomic_load_scalar`, guarded by `atomic_lowering` so it doesn't re-fire inside the RMW/CAS/store/load helpers): every lvalue-to-rvalue conversion of such an object passes through `gen_cast` *before* the `_Atomic` qualifier is stripped, so it lowers to `__atomic_load_<size>`. Verified on i386/arm under qemu; native 24/24 and conformance 70/70 (incl. the in-language read in `atomics.c`) green. ALSO DONE: in-language plain store to an _Atomic **aggregate or >8-byte object** (`_Atomic struct g; g = v;`, `_Atomic long double ld; ld = x;`) ‚ÄĒ previously a non-atomic struct/multi-word copy ‚ÄĒ now lowers to the generic `__atomic_store(size,&dest,&val,order)` libcall (`gen_atomic_store_aggregate`/`atomic_store_needs_generic`, hooked in `expr_eq`; needs `-latomic`). Verified vs gcc (24-byte struct + long double ‚Üí identical), references `__atomic_store` on all 5 arches; cli `atomic_inlang_aggregate_store`. RESOLVED N/A: **>8-byte fetch-ops** (`atomic_fetch_add` etc.) ‚ÄĒ mcc has no `>`word integer type (`__int128` unsupported) and fetch-ops on `float`/`long double` are invalid C (both mcc and gcc reject "integral or integer-sized pointer target type expected"), so there is nothing reachable to lower. ALSO DONE: in-language *read* of an _Atomic **aggregate / >8-byte object** (`Big r = g;`, `r = g;`, `*p = g;`) ‚ÄĒ the struct copy is taken from an atomic snapshot loaded via the generic `__atomic_load(size,&src,&tmp,order)` (`gen_atomic_load_aggregate`, hooked at the top of `vstore`, `atomic_lowering`-guarded). Verified vs gcc (`Big r1=g; r2=g;` ‚Üí identical `789 789`), references `__atomic_load` on all 5 arches; conformance 70/70 (vstore hot-path regression-checked). cli `atomic_inlang_aggregate` covers store+load. **In-language atomic load/store is now complete for scalars, pointers, and aggregates.** RMW on `long double`/`>`8-byte errors cleanly.
