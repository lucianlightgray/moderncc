FIXED 2026-07-22 (runtime/lib/armflush.c). arm64 `--embed-jit` baked output SIGSEGV'd at boot via infinite recursion; the runtime JIT is otherwise sound.

**Root cause:** `runtime/lib/armflush.c`'s aarch64 `__clear_cache(beg,end)` used, on the non-`__MCC__` (host gcc/clang) path, `__builtin___clear_cache(beg,end)`. On ELF AArch64 gcc AND clang lower that builtin to a **call to `__clear_cache`** — i.e. to the very function being defined — so `__clear_cache` self-recursed to a stack overflow. It's baked into `--embed-jit` output (via mccrt) and detonates the first time the JIT flushes the I-cache after writing code — inside the boot-time `mcc_jit_recompile_blob`, before any JIT-verbose output. Diagnosed by baking with `-debug=reloc` (sets `g_debug|=MCC_DBG_RELOC`, arm64-link.c:332 prints every CALL26 `name`+`val`) and finding the CALL26 whose `val==addr` (self): `name=__clear_cache`. (The classic "you can't implement __clear_cache with __builtin___clear_cache" footgun.)

**Fix:** replaced the `#else` builtin with the architectural clean+invalidate sequence inline (mrs CTR_EL0 → `dc cvau` loop → `dsb ish` → `ic ivau` loop → `dsb ish;isb`), matching libgcc/compiler-rt and mcc's own `__arm64_clear_cache`. Apple kept on `__builtin___clear_cache` (clang there lowers to `sys_icache_invalidate`, no self-call); `__MCC__` path unchanged (`__arm64_clear_cache` intrinsic). Verified: Linux/arm64 baked `hot.c` now boots the JIT (`mccjit-boot[sync]: ... blob len=1539 variant=... probe(7)=146`), no SIGSEGV, JIT-on==JIT-off results; native macOS `jit/selftest` 47/47 still green.

**Platform notes (still true):** running the JIT needs linux/arm64 (native under Docker on Apple Silicon) — bake first needs `-DCMAKE_C_FLAGS=-mno-outline-atomics` (else `__aarch64_ldadd8_acq_rel` unresolved at bake-link; mcc's linker has no libgcc). arm64 macOS still can't bake (Mach-O `_mccjit_boot_swap` unresolved + RWX gate). x86_64 JIT under Rosetta hangs. The JIT engine is proven in-process on arm64 macOS: `ctest -R jit/selftest` in cmake-build-embedjit = 47/47 (cross-session recompile, hotswap, spec/const-fold, KGC persistence, bench promote). Baking requires `-O1+` (not `-O0`) and is off with `-g`.
