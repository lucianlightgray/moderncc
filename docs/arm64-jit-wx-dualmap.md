**UPDATE (measured):** `host_runmem_dual()` is actually **false** on this host — `mmap(RW)` then `mprotect(RW→RX)` WORKS on Apple Silicon (only simultaneous RWX, and MAP_JIT without the codesign entitlement, are blocked). So the dual-map alias is NOT active and `host_runmem_alloc` returns a malloc buffer you must `host_runmem_protect(...,HOST_PROT_RX)`. The working stub pattern is **split-page**: `mmap(RW)` → write code → `host_runmem_protect(p, page, HOST_PROT_RX)` (also `__clear_cache`es). A *swappable* slot can't live in the RX code page (read-only) → put it in a **separate always-writable heap `void*`** and load the target absolutely: `movz/movk x17,<&slot>; ldr x16,[x17]; br x16`. Landed for the ptr-swap-slot hot-patch row (commit `01950395`, `jit/selftest-patch` shows it on arm64, faster than c-indirect). **[2B] remaining is coupled — partial work greens no test:** fix `mccjit_probe_exec_mem` (RWX→RW+mprotect-RX; else `mccjit_feasible()` stays false and the whole auto-dispatch layer is off), then port `mccjit_make_counter_stub`/`_kgc_stub_n`/`_kgc_stub_fp`/`_make_trampoline` + the `src/mccast.c` mode-6 slot, all split-page. The kgc-n per-arg marshalling (forward-order argv + `sxtw` narrow args) is the complex one; kgc-mixed has no qemu validation.

Original context follows. TODO **[2B]** (port the runtime-JIT dispatch/stub tail to arm64) has an Apple-Silicon complication the qemu validation did NOT cover — the qemu tests (`tests/qemu/jit_arm64_*.c`) ran under **Linux, where RWX mmap works**. On Apple Silicon macOS:

- Plain `mmap(PROT_READ|WRITE|EXEC, MAP_ANONYMOUS)` **fails** (verified). `MAP_JIT` + `pthread_jit_write_protect_np` **SIGBUS** without the `com.apple.security.cs.allow-jit` codesign entitlement (unavailable to `cc`-built test binaries). Only the **dual-map** works: `host_runmem_alloc` (`src/mcchost.c:1089`) maps `size*2` RX (`MAP_SHARED` on an `mkstemp`+`ftruncate` fd) then `MAP_FIXED`-remaps the 2nd half RW — write through `ptr+ptr_diff`, execute at `ptr`, `host_icache_flush` (=`__clear_cache`) the RX range. This is what makes the Stage-5 slice reemit execute on arm64.
- Consequence: `mccjit_probe_exec_mem` (`src/mccjit_embed.c:855`) uses RWX mmap → fails on arm64 → `mccjit_feasible()` false → the **entire runtime-JIT auto-dispatch/lazy layer is gated OFF** on this host. Direct `mcc_jit_recompile_blob` tests (kgc/stage2/leaf-int) pass; slot-dispatch tests (`lazy`/`pool`/`observability`/`liverun`) fail, and `fparg` fails at "fp-stub-swapped=0".
